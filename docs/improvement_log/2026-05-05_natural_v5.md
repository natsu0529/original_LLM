# 2026-05-05 v5: 多ターン合成 corpus を桁増やしして再学習

## 背景

v0.2.0 (`dazai-friend-natural-v4-768x12`, 87M params, val 1.466) を実走でユーザに使ってもらったところ、

- 「家にいるよ、あなたは？」→「特にこだわりないかも！」（topic mismatch）
- 「大丈夫だよ、少し遊ぼうよ」→「おいしそう。お腹すいてきた。」（off-topic）
- 「何を食べたの？」→「気になってることを、…」（前ターンの「お腹すいた」を全く受けていない）
- 「どゆこと？」→「引っ越したばっかで。早く買わないと。」（脱線）

…と、自由文の多ターン文脈追従が依然として弱いことが確認された。

原因仮説: 学習 corpus の **多ターン比率が低すぎる**。v4 corpus (chat_seed_friend_natural_v4) は約 33,000 ブロックのうち、明示的な多ターン block は v3 / v4 で追加した 2.8k 合成のうち 30 件程度しかない。`reply_loss_label=相手` は「直前の私: 入力に対する 相手: 返答」を学習するので、単一ターンに最適化されてしまう。

## 対処

### 多ターン合成を桁増やし

`scripts/synthesize_natural_chat_seed.py` に **`gen_parameterized_multi_turns`** を新設。直積展開で多ターンブロックを大量生成:

| カテゴリ | テンプレート構造 | おおよその件数 |
| --- | --- | --- |
| 食べ物（食べた／食べたい／飲み物） | 食べ物 × 3-4 ターン flow | ~150 |
| 場所 (PLACES × 3 turns) | 「{place}行きたい」→ 一緒に → 楽しみ | ~30 |
| 天気 | weather × 2 turns | ~12 |
| 趣味 | hobby × 2 turns | ~11 |
| 動物 | animal × 2 turns × 2 patterns | ~18 |
| 気分 (sad/happy) | mood × support × follow-up | ~30 |
| 睡眠／夢 | dream → repair → wake | 5 |
| 仕事／勉強 | work × follow-up | 6 |
| 主語ハイジーン (「私はXX」) | self_assert × 「そっちは？」 | ~6 |
| 「あなたは？」プローブ | you_probe × you_reply | ~12 |
| Repair turns | repair × ack | ~20 |
| 何してる？／何食べた？／今日何してた？ | probe × ack × ack | ~140 |
| 「遊ぼう」「一緒に」誘い | invite × follow | ~16 |
| えっと／うーん 躊躇 | hesitation × pull | ~16 |

合計約 **3,925 multi-turn ブロック**（synth 出力 v3）。

さらに `main()` の挿入比率を変更:

```text
Phase 1: すべての multi-turn block を一回ずつ書き出す
Phase 2: 残り単一ターン × 1 回 + multi-turn × 1 回 を 1:1 で交互（前: 1:5）
```

これで **multi-turn が dominant** な corpus に転換。

### 学習 corpus

`data/chat_seed_friend_natural_v5/`
- chat_seed_friend_clean_v2/ の 1668 ファイル
- chat_seed_friend_synth_v3/ の 246 ファイル（3925 ブロック）

合計 1914 ファイル / 約 1.91M chars。tokens 約 917k (train) + 115k (valid)。

### モデル/学習

v4 と同じ arch (87.5M params, n_layer=12, d_model=768, n_head=12, ffn_hidden=3072, context_length=512, vocab=3200)。
学習側は overfit 抑制を狙って **dropout 0.10 → 0.15**, **weight_decay 0.05 → 0.08** に強化。LR / cosine / warmup は同じ。

```
batch_size=8 max_steps=6000 lr=4e-4 → 4e-5 (cosine)
warmup_steps=250 weight_decay=0.08 dropout=0.15
```

## 結果

eval 履歴抜粋:

| step | train | valid | best |
| --- | --- | --- | --- |
| 200 | 3.15 | 3.09 | 3.09 |
| 600 | 1.41 | 1.93 | 1.93 |
| 1000 | 1.36 | 1.67 | 1.67 |
| 1200 | 1.17 | 1.55 | 1.55 |
| 1400 | 1.31 | **1.454** | **1.454** |
| 3600 | 0.33 | **1.441** | **1.441** ← best.pt 確定 |
| 6000 | 0.06 | 1.87 | 1.441 |

best step = 3600, **valid_loss = 1.4414** （v4 best 1.466 から 1.7% 改善）。step 3600 以降は train 急降下/valid 不安定の overfit ゾーン。

| 何してる？ | ぼんやりしてる。少し話せるよ。 | ぼんやりしてる。少し話せるよ。 |

| 映画でも見ようかな | おすすめあるなら教えて。 | おすすめあるなら教えて。 |

multi-turn 系の improvement は出ているが、自由文の coherence は依然として model キャパが律速。期待した「家にいるよ、あなたは？」→「私は本読んでた」のような subject-aware reply はまだ。

## 残課題と仮説

- 87M params + 1.9M chars corpus は overfit が早い（step 3600 で best、以降 train 0.06 まで落ちて valid は劣化）。
- 「私は X」入力を見たときの「相手は Y」の subject hygiene は学習されているが、retrieval / curated の優先順序より弱い。
- **次の方向性**:
  1. corpus 量の増加（青空文庫からの自然対話抽出を追加 → 4-5 倍）
  2. dropout 0.2 / 学習 step 8000 + early stop
  3. context_length 768 へ拡張（multi-turn 履歴が 2 ターン以上保持できる）
