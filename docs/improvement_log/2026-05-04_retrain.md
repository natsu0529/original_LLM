# 2026-05-04 大きいモデル + 拡張 corpus で再学習

## 出発点（昨日からの引き継ぎ）

[2026-05-03_naturalness.md](./2026-05-03_naturalness.md) で runtime 側のヒューリスティックを揃え、curated 短文返答とフィルタで挨拶〜気分系の入力はかなり自然になった。
ただし `映画でも見ようかな` `ピクニック行きたいな` のような自由文に対しては、モデル本体（dazai-friend-topic-clean-spm-384x6-v3, ~4M params, step 1550, val_loss=1.18）が壊れた日本語を出すことが残課題だった。

ユーザから「再学習・データ追加・アーキ変更を許可、tmux も使ってよい」と明確な合意を取った。

## 観察された問題

- 既存 corpus `chat_seed_friend_clean_v1` には自動生成由来のノイズ（`つらい` → `もどすのが。`, `楽しい` → `オーブンがない！`, `お腹すいた` → `食べたいない…` など）が多数残存している。
- 学習量も少ない（step 1550）。モデルキャパも小さい（d_model=384, n_layer=6 ≒ 4M params）。
- 結果として novel な自由文では mode collapse 寄り＋文法破綻寄りの返答になる。

## 推定原因

1. **データの質**: 雑談の自然さに直結する「日常トピック × 短い相づち / 提案 / 同意」の seed が薄い。
2. **データの量**: 自然な雑談にラベル付きで触れる経験が足りない。
3. **モデルキャパ**: 4M params では多様な日常フレーズの分布を覆いきれない。

## 対処

### 1. 高品質な合成会話 seed を追加

`scripts/synthesize_natural_chat_seed.py` を新規追加。

- 食べ物（食べた / 食べたい）、飲み物、場所、天気、映画 / ピクニック / ドライブ、趣味、動物、気分（落込・うれしい）、睡眠、仕事 / 勉強、挨拶、短文相づちのカテゴリで `私: / 相手:` ペアを生成。
- 各カテゴリで `_product_pairs(user_templates, reply_templates, slot_items, key)` の直積で展開。
- マルチターン会話（2 ターン）も少量混ぜる。
- 出力: `data/chat_seed_friend_synth_v1/` に 160 ファイル, 約 2,500 ブロック。

### 2. v2 (低品質除去済) と合成 seed をマージ

```
data/chat_seed_friend_natural_v3/  (1828 .txt)
  - data/chat_seed_friend_clean_v2/ の 1,668 ファイル
  - data/chat_seed_friend_synth_v1/  の 160 ファイル
合計 ≒ 1.8M chars / sentencepiece tokens 約 893k (train) + 113k (valid)
```

### 3. `train.py` に LR スケジューラを追加

cosine 減衰 + linear warmup を追加：

- `--lr-schedule cosine`
- `--warmup-steps 200`
- `--min-learning-rate 5e-5`

`compute_lr` / `set_optimizer_lr` を関数化し `tests/test_train_lr_schedule.py` で 4 ケースをテスト（warmup 中の線形性、cosine の終端 / 中点、constant の不変性）。

### 4. アーキ拡大 + 再学習

新 run: `dazai-friend-natural-v3-512x8`

- d_model=512, n_layer=8, n_head=8, ffn_hidden=2048, context_length=256, dropout=0.1
- 約 26.6M params (旧モデルの ~6 倍)
- vocab_size=2400 (sentencepiece, 既存より少し広い)
- batch_size=16, grad_clip=1.0, weight_decay=0.05
- learning_rate=5e-4 → 5e-5 (cosine), warmup_steps=200
- max_steps=6000 (≒ 44 分 on M4 MPS)
- reply_loss_label=相手 で reply 部分にだけ loss
- `caffeinate -dimsu tmux new-session -d -s llm-train PYTHONUNBUFFERED=1 python -u src/train.py …` で背景起動

> 1 度目の起動は `python | tee log` の段階で stdout が block-buffered になり log がしばらく空だった。
> `PYTHONUNBUFFERED=1` + `python -u` + `> log 2>&1`（tee なし）に切り替えて解消。

### 5. 推論側の優先度を更新

- `preferred_chat_checkpoint()` の glob pattern 先頭に `dazai-friend-natural-v3-*/best.pt` と `dazai-friend-natural-*/best.pt` を追加。
- `resolve_retrieval_corpus_dir()` の bundled 候補先頭に `data/chat_seed_friend_natural_v3` を追加。
- `tests/test_update_check.py::test_resolve_retrieval_prefers_bundled_chat_seed_over_raw_checkpoint_dir` を「推奨ディレクトリの順序を hard-coded ではなく実際に存在する一番上を取る」形に書き換えた。

## 学習結果（最終）

| step | train_loss | valid_loss | best_valid_loss | 備考 |
| --- | --- | --- | --- | --- |
| 1 | 7.88 | 7.89 | 7.89 | warmup 開始時 |
| 200 | 2.55 | 2.54 | 2.54 | warmup 終了 |
| 800 | 1.36 | 1.61 | 1.61 | |
| 1000 | 1.25 | 1.54 | 1.54 | |
| 2000 | 0.87 | 1.50 | **1.50** | best.pt はここで確定 |
| 3000 | 0.41 | 1.60 | 1.50 | 過学習が始まる |
| 4000 | 0.17 | 1.81 | 1.50 | train が drop し続ける |
| 6000 | 0.07 | 1.76 | 1.50 | 完了 |

step 2000 を超えると train は下がるが valid が上下するので、`best.pt` は step 2000 のもの。実走に使うのはこれ。
旧 checkpoint (`dazai-friend-topic-clean-spm-384x6-v3`) の best_val_loss=1.18 と単純比較はできない（学習 corpus が違う）が、検証 corpus は新しい方が広く・難しいことを踏まえれば 1.50 は同等以上。

学習所要: ~47 分（M4 MPS）。

学習途中の生成例（step 6000 sample）:
```
私: こんにちは
相手: 明日のご飯のご定は？  # 「予定」を「ご定」と崩す等の小さな破綻はあるが、文脈は追えている
```

## 推論側に効かせた変更

- `preferred_chat_checkpoint()` の glob 先頭に `dazai-friend-natural-v3-*/best.pt` を追加 → `dazai-chat` を引数なしで叩くと自動でこの新 checkpoint が選ばれる
- `resolve_retrieval_corpus_dir()` の bundled 候補先頭に `data/chat_seed_friend_natural_v3` を追加
- 既存テストは新優先順を踏まえて書き換え。`uv run python -m pytest tests/` 全 42 件パス。
