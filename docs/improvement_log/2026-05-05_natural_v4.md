# 2026-05-05 v4: 87M params / context 512 / multi-turn 強化

## 動機

[2026-05-05_prompt_tuning.md](./2026-05-05_prompt_tuning.md) 直後の実走で、自由文の文脈追従弱さが残った（夢の話がループ、主語逆転、`どういうこと？` の未知語誤検出など）。
ユーザから「16 GB を最低スペックと前提していい、モデル能力を底上げしてよい」と合意があったので、**コンテキスト 256 → 512、26.6M → 87M params** にスケールアップ。

## 実装

### 合成 corpus の拡張

`scripts/synthesize_natural_chat_seed.py` の `gen_two_turn_followups` を拡張:

- 朝起き直後の 3 ターン flow（`よう` → `寝れた` → mood probe）
- 夢の話を 1〜2 ターンで終わらせるブロック
- 「何言ってるのさ」「意味わかんない」「どういうこと？」などの **repair turn**
- 「私はもう起きたよ」「私はラーメン食べた」のような **明示的な主語入力に対する subject hygiene**
- 「今日は何してた？」が来た時に **自分の話を返す** flow

multi-turn の挿入頻度も 1/10 → 1/5 に上げ、不足時は追加で 3 周ループ。
新 dir: `data/chat_seed_friend_synth_v2/`（180 ファイル / 2868 ブロック）

merge: `data/chat_seed_friend_natural_v4/`（1848 ファイル / 約 1.84M chars）

### アーキ拡張

| 項目 | v3 | v4 |
| --- | --- | --- |
| n_layer | 8 | **12** |
| d_model | 512 | **768** |
| n_head | 8 | **12** |
| ffn_hidden | 2048 | **3072** |
| context_length | 256 | **512** |
| vocab_size | 2400 | **3200** |
| dropout | 0.1 | 0.1 |
| **params** | 26.6M | **87.4M** |
| **slim ckpt size** | 107 MiB | **344 MiB** |

学習設定: `lr=4e-4 → 4e-5 cosine`, `warmup=250`, `weight_decay=0.05`, `batch_size=8`, `max_steps=6000`。

### 運用上の落し穴

1. **disk full crash** (step 1400 で valid 1.756 を best 確定後に落ちた)
   - 原因: 1 GB 超の checkpoint を `step_*.pt` として 500step ごとに書いていた
   - 対処: `--no-step-checkpoints` を付けて resume、古い run の checkpoints (~10 GB) を削除

2. **tmux session の cwd**
   - resume 時、`tmux new-session` の cwd が前の `cd` 履歴で `checkpoints/` のままだったため `python -u src/train.py` が見つからず即落ち
   - 対処: `tmux new-session -c /Users/.../original_LLM` と絶対パス指定

3. **cli.py / pyproject.toml の手動修正**
   - ユーザが `[project.scripts]` から `original-llm` 行を消していた（`dazai-chat` のみに絞る方針）
   - 反映を尊重して PR 等は出さず継続

## probe（`uv run dazai-chat --checkpoint .../v4/best.pt --seed 7`）

| 入力 | v4 出力 | 評価 |
| --- | --- | --- |
| こんにちは | こんにちは。今日はどうしてた？ | ✓ curated |
| よう | おはよう。今日はゆっくりやろう。 | ✓ curated |
| うん、寝れた | そういう日は、むずかしい話をしないほうがいい。 | × off-topic |
| 私はもう起きたよ | おはよう。まだ頭が少し白いだろう。 | × subject 主語逆転は減ったが新たな不自然 |
| 何言ってるのさ | そんなにあっさり答えられる話でもないよ。少し考えさせてくれ。 | △ |
| どういうこと？ | サザンの歌。涙のキスもう一度。 | × off-topic（ただし v3 の未知語誤検出は無くなった、conf が上がったから） |
| 今日は何してた？ | ごめん、変な言い方だった。もう少し普通に言うね。 | △（repair seed 由来） |
| いい気分？ | いいと思う。少し頼りないけれど、動く。 | × |
| 今日疲れた / お腹すいた / Hello | curated/guard で正解 | ✓ |
| 映画でも見ようかな | おすすめあるなら教えて。 | ✓ retrieval/synth 由来 |
| ピクニック行きたいな | サンドイッチ作っていく？ | ✓ |
| ぴえん | 引っ越したばっかで。早く買わないと。 | × unknown 検出から漏れる（conf 高め）|
| クリプトコイン | クリプトコインか、教えてもらってもいい？ | ✓ unknown 検出 |
| アップデートのやり方 | (curated) | ✓ |

**所感**:

- 既存の curated / guard / synth 経由の入力は正解率高い（v3 と同等以上）
- novel 自由文の文脈追従はまだ弱い。multi-turn 合成データは 30k のクリーン corpus に対して 2.8k と相対量が小さく、効果が薄かった
- 合成 corpus を **メイン corpus と同等の量**（10k-30k ブロック）で揃えないと multi-turn 学習は浸透しない
- ぴえん は v3 と同じくモデルが一見もっともらしく続けてしまう mode collapse（memory に教えれば対応可）

## 次の段階の選択肢

1. **合成 multi-turn を ×10 に増やす + dropout 0.2 へ**: 一番効果が読める
2. **Gradient accumulation で実効 batch=32 へ**: 学習安定化
3. **学習 corpus と retrieval corpus を分離**（学習は synth 多め、retrieval は cleaned 多め）

このリリースには含めず、v0.2.x の段階で詰める。
