# 2026-05-05 セッションをまたぐローカル記憶（SQLite）

## ねらい

ChatGPT / Gemini のブラウザ版が持つ「会話本文は覚えていないが、ユーザの肩書き・興味・名前などは別セッションでも覚えている」という挙動を、サーバなしの CLI で再現する。

## 設計（採用案）

ユーザ提案: **1 テーブル丸ごと記憶テーブル / `id, key, value, 時刻, 重要度`**

これを採用。スキーマは:

```sql
CREATE TABLE memory (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  key         TEXT NOT NULL,
  value       TEXT NOT NULL,
  importance  INTEGER NOT NULL DEFAULT 3 CHECK (importance BETWEEN 1 AND 5),
  created_at  TEXT NOT NULL,
  updated_at  TEXT NOT NULL
);
CREATE INDEX idx_memory_imp_updated ON memory(importance DESC, updated_at DESC);
CREATE INDEX idx_memory_key ON memory(key);
```

- `key` は自由文字列（`name` / `専攻` / `ペット` / `note` / `interest:猫` など何でも可）
- `key` は UNIQUE にしていない。同じ `name` を新たに追加するとレコードが増える。重複したくなければ `:forget-key name` で消してから追加する運用。
- `importance` は 1〜5。挿入時に範囲外を投げると 1 か 5 にクランプ。
- `created_at` / `updated_at` は ISO-8601 / UTC。

格納先（XDG 準拠）:

```
$XDG_DATA_HOME/original-llm/memory.db
# fallback
~/.local/share/original-llm/memory.db
```

`$ORIGINAL_LLM_MEMORY_DB` で完全に上書き可。CLI からは `--memory-db PATH` で 1 セッションだけ別ファイルにできる。

## CLI 追加機能

interactive モードに以下のスラッシュコマンドを追加。

| コマンド | 動作 |
| --- | --- |
| `:memory` | 全件を `importance DESC, updated_at DESC` で表示 |
| `:memory clear` | 全消去 |
| `:remember <key> <value>` | 既定 importance=3 で追加 |
| `:remember <key> <value> 5` | 末尾に 1〜5 を渡すと重要度指定 |
| `:remember key=value` | `=` 形式もサポート |
| `:forget <id>` | id 指定で 1 件消す |
| `:forget-key <key>` | 同 key を全削除 |

`dazai-chat` 起動時にエントリ数とパスを表示:

```
memory-db: /Users/.../memory.db (3 entries)
```

`--use-memory` / `--no-use-memory` で機能 ON/OFF。`--memory-inject N` で prompt 先頭に注入する件数（既定 3）。

## prompt への注入

interactive_loop の model 経路（curated/直接 reply には不要）で、入力を準備した直後に:

1. `MemoryStore.select_relevant(prepared_user_input, limit=memory_inject_limit)` で重要度 + recency + 入力文との部分一致をスコアして上位 N 件を取得。
2. 1 つの chat turn にまとめて prepend:

```
私: 覚えておいてね: nameは夏樹、専攻は計算機科学。
相手: うん、覚えた。
<eot>

私: <user input>
相手:
```

3. context_length=256 を超えないよう `trim_text_to_context` で切り詰める。
4. retrieval_block / history はその後ろに来るので、最重要の memory は前段で生き残る。

curated_short_reply / select_direct_chat_reply は元のまま動く（モデル生成しないので memory 注入の影響を受けない）。

## relevance スコア

```python
score = entry.importance * 10
+ 50 if entry.key in user_input
+ 30 if entry.value in user_input
+ 1 per matching char in entry.key
```

短文／日本語前提で、tokenizer フリーに動く。query が空なら `importance DESC, updated_at DESC` の上位を返す。

## プライバシー

- DB は完全にローカル。ネットワーク送信なし（モデル本体のダウンロード以外）。
- `:memory clear` で完全消去できる。
- DB ファイルを別マシンに rsync すれば移植可能。
- 過去会話の本文は **保存しない**（key/value で抽象化された事実だけ）。

## 検証

```bash
TMP_DB=$(mktemp); rm -f "$TMP_DB"

printf ':remember name 夏樹 5\n:remember 専攻 計算機科学 4\n:memory\n:quit\n' \
  | uv run dazai-chat --memory-db "$TMP_DB"

# プロセスを抜けてもう一度開く → 記憶が残っている
printf ':memory\n今日も研究頑張る\n:quit\n' \
  | uv run dazai-chat --memory-db "$TMP_DB" --show-prompt-output
```

`[prompt]` ブロックに

```
私: 覚えておいてね: nameは夏樹、専攻は計算機科学。
相手: うん、覚えた。
<eot>

私: 今日も研究頑張る
相手:
```

が出ていれば成功。出力例: `相手: すごいな。今日はどんな感じとか？`

## 今後の拡張余地

- 暗黙抽出（`私の名前は…` のような入力から正規表現で `name` を取り出して y/n 確認）
- 重要度の自動減衰（古い fact を `importance -= 1` にする）
- 全文検索（FTS5 を使った値検索）
- export/import JSON（環境を移すとき）
- `:bump <id>` / `:lower <id>` で重要度を上下させる UI

これらは段階的に積めるが、今回は 1 テーブル単純構成で「セッションをまたいで覚える」最低ラインを達成した。
